基础常识

磁盘

1.寻址：ms
2.带宽：G/M
##内存
1.寻址：ns
2.带宽：很大
时间单位
秒>毫秒>微妙>纳秒
内存寻址比硬盘寻址快10万倍
I/O buffer 成本问题
磁盘与磁道、扇区，一扇区512byte,带来的成本变大：索引
4K对齐，操作系统，无论读多少都是最少4K从磁盘拿
随着文件变大，速度变慢，IO成为瓶颈

数据库

data page 4k
关系型数据库建表： 必须给出schema
类型： 建表宽度
存：倾向于行级存储
内存维护一个B+TREE
磁盘：存储数据和索引
表很大的时候，性能下降？
1.如果有索引，增删改变慢
2.查询速度
2.1如果一个或少量查询，性能依然快，因为where条件走的依然是一个data page
2.2当并发量来了，可能不是加载一个data page到内存，受硬盘带宽影响，有可能性能下降(IOPS)
